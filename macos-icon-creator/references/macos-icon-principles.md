# macOS Icon Principles

Use these rules as operating guidance, not as prose to repeat back to the user.

## Core goal

Translate the app idea into a native-feeling object or scene. The icon should feel designed for the macOS dock, not adapted from a brand logo.

## Metaphor selection

- Start from the app's main noun, verb, or emotional promise.
- Prefer one primary metaphor with one supporting cue.
- Pick a metaphor with a bold silhouette before thinking about detail.
- If the concept is abstract, turn it into a tangible object, instrument, tool, surface, or layered symbol.

## Composition

- Use the squircle as a base geometry the icon can fuse with, sit on, or push against.
- Favor one dominant focal object.
- Support it with one base plane, halo, inset, or secondary layer if needed.
- Let the object push toward an edge when that adds energy, but do not crop away recognition.
- Allow secondary pieces to protrude beyond the squircle when the breakout improves the metaphor and remains clean at small sizes.
- Avoid the “everything lives safely inside the rounded square” composition unless the concept specifically wants containment.
- Keep the icon readable as a single silhouette from a distance.

## Material and depth

- Prefer believable material logic: glass, enamel, machined metal, paper, translucent plates, soft glows.
- Use gradients to explain light and curvature, not to decorate empty space.
- Add highlights only where a plane actually turns toward the light or where a material needs a specular cue.
- Skip highlights when the form already reads through silhouette, overlap, inset depth, or tonal separation.
- Add shadows where forms overlap or float.
- Keep filter usage light; favor vector geometry and tonal contrast over blur-heavy effects.

## Native-feeling constraints

- Avoid a centered logo sitting on a plain rounded square.
- Avoid the generic "top sheen" treatment unless the icon truly has a glossy surface.
- Avoid treating the squircle as a blank sticker behind the icon.
- Avoid screenshot fragments, windows, paragraphs of text, or tiny interface chrome.
- Avoid too many equally important objects.
- Avoid details that disappear below 64 px.
- If a brand color exists, integrate it into the metaphor rather than letting it become the whole icon.

## Small-size heuristics

- At 16 px, the main silhouette should still read.
- At 32 px, the main object and one supporting cue should remain distinct.
- At 64 px, depth should be visible without requiring texture.
- At 128 px, the icon can reveal secondary material treatment, but the composition must still feel simple.

## Preferred workflow

1. Identify the primary metaphor.
2. Choose one composition family.
   - fused-with-base
   - perched-on-stage
   - breakout-corner
   - inset-object-with-overlap
3. Decide background plane and palette.
4. Add depth and lighting.
5. Remove weak detail until the icon still reads small.
